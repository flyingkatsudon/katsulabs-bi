# 목표 아키텍처 (Clean Architecture)

## 레이어

```
┌─────────────────────────────────────────────────────────┐
│  api (Spring Boot 4, REST, Security)                  │
│  - Controller, Config, @SpringBootApplication           │
└──────────────────────────┬──────────────────────────────┘
                           │ depends on
┌──────────────────────────▼──────────────────────────────┐
│  infrastructure                                         │
│  - MyBatis/JPA, Redis, DataProvider adapters            │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  application                                            │
│  - Use cases, Port interfaces (in/out)                  │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  domain                                                 │
│  - Entities, Value Objects, Domain exceptions           │
│  - NO Spring, NO framework imports                      │
└─────────────────────────────────────────────────────────┘
```

## 디자인 패턴 (실용적 적용)

| 패턴 | 적용 위치 | 이유 |
|------|-----------|------|
| **Repository** | domain 포트 + infrastructure 구현 | 영속성 교체 |
| **Strategy** | `DataProvider` 타입별 조회 | JDBC/Kylin/ES 등 |
| **Factory** | 차트·집계 설정 생성 | 복잡한 config JSON |
| **Adapter** | 레거시 DTO ↔ domain | 점진 이전 |
| **Template Method** | 공통 쿼리 실행 흐름 | 중복만 줄일 때 |

**지양**: 모든 서비스에 Facade/AbstractFactory, 1:1 Interface 남발

## JSON

- 애플리케이션 내부: `org.cboard.application.support.JsonMapper` (Jackson)
- REST: Spring Boot 기본 Jackson 직렬화
- Fastjson / `org.json` 사용 금지 (신규 코드)

## TDD

1. **domain / application**: JUnit 5 + 순수 단위 테스트 (Mock 없이 가능한 범위)  
2. **api**: `@WebMvcTest`, `@SpringBootTest` + Testcontainers(Postgres)  
3. **frontend**: Vitest + React Testing Library  

테스트 명명 (한글 display name 허용):

```java
@DisplayName("헬스 체크는 UP 상태를 반환한다")
@Test
void healthReturnsUp() { ... }
```

## 프론트엔드

```
frontend/
├── src/
│   ├── app/          # 라우트, 레이아웃
│   ├── features/     # 기능 단위 (dashboard, dataset, …)
│   └── shared/       # API client, UI primitives
└── public/
    └── styles/       # 기존 AdminLTE CSS (복사·참조)
```

## 환경 프로필

| 프로필 | DB | 용도 |
|--------|-----|------|
| `local-h2` | H2 file | 빠른 로컬·단위 통합 |
| `local-postgres` | PostgreSQL (Docker) | 운영에 가까운 로컬 |
| `docker` | MySQL (기존 compose) | 레거시 호환 기간 |

## API 버전

- `/api/v1/...` — 신규 REST
- 레거시 `/bdp/*` — Strangler 기간 동안 프록시 또는 병행
