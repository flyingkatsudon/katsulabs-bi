## 레거시 전용 차트 정책

React 뷰어(`WidgetChartView`)는 **ECharts 기반** 차트만 렌더합니다.  
`CHART_LEGACY_ONLY` (`frontend/src/utils/chartRender.ts`)에 해당하는 유형은:

| 유형 | 미이식 이유 (요약) |
|------|-------------------|
| `googleMap`, `map`, `chinaMap`, `areaMap`, … | Google/Baidu/지도 SDK·API 키·레거시 Angular 지도 위젯 의존 |
| `fusionganttcharts` | FusionCharts 상용/구버전 JS 번들 |
| `wordBubble`, `liquidFill`, `heatMapCalendar`, `relation` | 레거시 전용 플러그인 또는 ECharts 옵션 미구현 |

이들은 **집계 API는 동일**하지만, 뷰어는 표로 대체하고 위젯 편집 화면에서 설정을 유지합니다. 이식하려면 차트별로 React+ECharts(또는 Map 라이브러리) **신규 구현**이 필요합니다.

## 정적 UI 자산 (「레거시 자산 동기화」 설명)

- **의미**: 예전 CBoard WAR의 `webapp` 안 AdminLTE·Font Awesome·플러그인 CSS/JS를 `frontend/public/insightboard/`에 두고 쓰는 방식입니다.
- **현재**: 자산은 **이미 저장소에 포함**되어 있으며, 개발 시 별도 동기화 없이 Vite가 `/insightboard/` 로 서빙합니다.
- **동기화 스크립트**: `frontend/docs/UX-STRATEGY.md`에 언급된 `sync-cboard-assets.sh`는 **아직 없음**. upstream CBoard에서 CSS를 다시 가져올 필요가 생기면 그때 스크립트를 추가하면 됩니다. 일상 개발에는 **필수 작업이 아닙니다**.

## 로컬 실행

```bash
chmod +x scripts/run-api.sh scripts/stop-dev.sh
./scripts/run-api.sh
cd frontend && npm install && npm run dev
```

- UI: http://localhost:5173 — **admin01** / **admin123**
- Postgres: `docker compose -f docker-compose.postgres.yml up -d` 후 `local-postgres` 프로필
