# CBoard 사용법

> 대상: **신규 Boot 4 API** (`modules/insight-board-api`) 및 **레거시 WAR** (`mvn tomcat7:run`)  
> JSON 처리: **Jackson** (`org.cboard.application.support.JsonMapper`)

## 1. 실행 방법

### 신규 스택 (권장 — 로컬 개발)

```bash
# 1) 테스트·DB 마이그레이션 확인
./gradlew test

# 2) H2 메모리 DB로 API 기동 (Flyway 자동 적용)
./gradlew :modules:insight-board-api:bootRun --args='--spring.profiles.active=local-h2'

# 3) PostgreSQL (선택)
docker compose -f docker-compose.postgres.yml up -d
./gradlew :modules:insight-board-api:bootRun --args='--spring.profiles.active=local-postgres'
```

- 기본 URL: `http://localhost:8081`
- H2 콘솔: `http://localhost:8081/h2-console` (JDBC URL은 `application-local-h2.yml` 참고)

### 레거시 WAR (AngularJS UI)

```bash
docker compose up -d
mvn clean package tomcat7:run -Denv=docker
```

- URL: `http://localhost:8080/bdp/`
- 로그인 JSP → `cboard/starter.jsp` 대시보드

### 프론트 (React + Vite)

```bash
cd frontend && npm install && npm run dev
```

- `http://localhost:5173` — `/api` 는 Boot `8081` 로 프록시
- **UI/UX (기본)**: React 구현 + 레거시 CSS/메뉴/상세 UX — Boot API 만으로 실행. → [frontend/docs/UX-STRATEGY.md](../frontend/docs/UX-STRATEGY.md)
- **UI/UX (iframe)**: `VITE_USE_LEGACY_UI=true` — 예전 Angular `starter.jsp` 전체 (Tomcat 필요)

## 2. 로그인

### 신규 REST API

```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "userId": "admin01",
  "password": "admin123"
}
```

`businessCode` 를 생략하면 `cboard.default-business-code` (기본 `SH`) 가 사용됩니다.

**성공 응답 예:**

```json
{
  "userId": "admin01",
  "businessCode": "SH",
  "loginName": "admin01",
  "displayName": "Admin",
  "roleId": "1",
  "resourceTypes": [1]
}
```

세션 쿠키가 발급되며, 이후 API 호출 시 동일 브라우저/세션을 유지해야 합니다.

**curl 예:**

```bash
curl -c cookies.txt -X POST http://localhost:8081/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"userId":"admin01","password":"admin123"}'

curl -b cookies.txt http://localhost:8081/api/v1/boards
```

### 레거시 폼 로그인

- URL: `POST /bdp/process`
- 필드: `v0` = business_code (hidden), `v1` = 사용자 ID, `v2` = 비밀번호
- 기본 계정 (Docker 시드): `admin01` / `admin123`, business `SH`

## 3. 대시보드 API (신규)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/v1/health` | 헬스 (인증 불필요) |
| GET | `/api/v1/categories` | 카테고리 목록 |
| POST | `/api/v1/auth/login` | 로그인 |
| GET | `/api/v1/boards` | 보드 목록 |
| GET | `/api/v1/boards/{id}` | 보드 상세 (`layoutJson` 포함) |
| POST/PUT/DELETE | `/api/v1/boards`, `/api/v1/boards/{id}` | 보드 생성·수정·삭제 |
| GET | `/api/v1/datasets` | 데이터셋 목록 |
| GET | `/api/v1/datasets/{id}` | 데이터셋 상세 (`dataJson`) |
| POST/PUT/DELETE | `/api/v1/datasets`, `/api/v1/datasets/{id}` | 데이터셋 CRUD |
| GET | `/api/v1/datasets/{id}/preview?limit=50` | 데이터셋 SQL 미리보기 (JDBC) |
| GET | `/api/v1/datasources` | 데이터소스 목록 |
| GET | `/api/v1/datasources/{id}` | 데이터소스 상세 (`configJson`) |
| POST/PUT/DELETE | `/api/v1/datasources`, `/api/v1/datasources/{id}` | 데이터소스 CRUD |
| POST | `/api/v1/datasources/test` | 저장 전 JDBC 연결 테스트 (body = 등록과 동일) |
| POST | `/api/v1/datasources/{id}/test` | JDBC 연결 테스트 |
| GET | `/api/v1/widgets` | 위젯 목록 |
| GET | `/api/v1/widgets/{id}` | 위젯 상세 (`dataJson`) |
| POST/PUT/DELETE | `/api/v1/widgets`, `/api/v1/widgets/{id}` | 위젯 CRUD |
| POST | `/api/v1/aggregate` | 집계 쿼리 (레거시 `DataProviderService` 연동 후 활성화) |

레거시 AngularJS 가 호출하던 `/bdp/cboard/dashboard/*` 와 **경로가 다릅니다**. React 마이그레이션 시 위 API를 사용하세요.

## 4. JSON 처리 (Jackson)

애플리케이션 코드에서는 **Fastjson 대신** `JsonMapper` 를 사용합니다.

```java
import org.cboard.application.support.JsonMapper;

String json = JsonMapper.toJson(dto);
MyDto dto = JsonMapper.fromJson(json, MyDto.class);
```

Spring MVC REST 응답은 Boot 기본 Jackson이 직렬화합니다. DTO ↔ 도메인 변환·레이아웃 JSON 파싱 등은 `JsonMapper` 로 통일합니다.

## 5. CBoard UI 개념 (레거시)

1. **DataSource** — JDBC/Kylin/ES 등 외부 연결
2. **Dataset** — 차원·지표 정의 (JSON)
3. **Widget** — 차트/테이블 위젯 정의
4. **Board** — 위젯 배치(`layout_json`)가 있는 대시보드
5. **Category** — 폴더 분류

작업 흐름: DataSource 생성 → Dataset 설계 → Widget 연결 → Board에 배치.

신규 API는 **CRUD** 및 세션 로그인을 제공합니다. **집계**(`/api/v1/aggregate`)는 레거시 엔진 브리지 완료 후 `@ConditionalOnBean` 으로 활성화됩니다.

**OpenAPI:** `http://localhost:8081/swagger-ui.html`

**쓰기 응답** (`ServiceResult`): `status` `"1"` 성공, `"2"` 실패, `message`, 선택적 `id`.

## 7. 문제 해결

| 증상 | 확인 |
|------|------|
| 401 로그인 실패 | Flyway 시드 적용 여부, `admin01`/`admin123`, business `SH` |
| 빈 보드 목록 | `V2__seed_dev.sql` 의 Demo Board 존재 여부 |
| 레거시 Driver 오류 | `mvn package` 후 WAR에 JDBC 드라이버 포함 여부 |
| React API 연결 실패 | Boot 8081 실행, Vite 프록시 설정 |
